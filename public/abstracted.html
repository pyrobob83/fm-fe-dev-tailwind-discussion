Same setup as before, but using abstraction for:

  • `space-vert` class applies vertical spacing that changes between responsive steps
  • `block` class applies TW's `container` class for max-width, `mx-auto` to center the div in the viewport and `px-8` to add horizontal padding to ensure content never touches the edge of the device
  • `bordered` class to apply the block borders that help illustrate

This is a typical example of a container how I'd normally use it (referred to as a 'block'):

  • inside a 'section' that has a BG color (or image) + some vertical padding applied to the section
  • container has horizontal padding (px-8) to keep content indented at smaller screens
  • container also has it's own vertical padding for vertical space between 'blocks' inside the section
  • Result is 2 sets of vertical padding within the overall 'section' above + below the content

The downside of this method is that you can't ever have two 'blocks' appear to be inside the same section - for example if you wanted a background image on the 'section' but with varying 'blocks' inside of it.
